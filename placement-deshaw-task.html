Empowering traders with data vis

Analyzing data and predicting the future from it can get tedious if the data is consumed in a raw tabular form. How might we then make data consumption more efficient to enable traders achieve their goals?

This project was done in 2018 as a part of "D.E. Shaw India Placement Task" in a duration of 24 hours.

Project Overview

'InVis' is a data visualization interface for traders which enables effective and meaningful analysis- both consumption and production- of data.

Design Process

I followed the What-Why-How framework for Data Visualization Design proposed by data vis guru Prof. Tamara Munzner.

What: Data Abstraction

“Understanding the given dataset”

Many aspects of vis design are driven by the kind of data at our disposal. So, I started to gather sense of the mammoth data set given to us. The framework allowed me to do exactly that quite efficiently-

Understanding the data-set

“Understanding traders using context-scenario”

To understand how a trader might use the given data set, I drew up a story board highlighting a context of use. This allowed me to elicit the goals of a trader.

How might a trader use this dataset

Why: Task Abstraction

“Eliciting user goals as abstract tasks”

The most important part of this framework is to consider user goals (obtained from the scenario above) in an abstract form. They are called “visualization tasks”. These tasks were fundamental in concisely identifying the underlying user goals which guided my design of vis and other features in the interface.

Listing down visualization tasks from user goals.

How: Design vis idioms

“Encoding data into visualizations"

After distilling tasks from goals, I ideated upon different visualizations that could support those goals. The "How" framework was particularly helpful while designing visualizations. The elements of the framework that I used for my design decisions are as mentioned in the figure.
